## UNIVERSAL NOTIFICATION COLORS
*These colors remain CONSISTENT across all theme modes*

### System Notification Color Palette:
- **Success:** #00C851 (Green)
  - Usage: Order confirmations, successful actions, completed states
  
- **Error:** #FF4444 (Red)  
  - Usage: Form validation errors, failed transactions, alert messages
  
- **Warning:** #FFD700 (Yellow)
  - Usage: Low stock alerts, pending actions, caution states
  
- **Info:** #6589FF (Blue)
  - Usage: Help tooltips, information messages, neutral notifications
  
- **Rating:** #FF6740 (Orange)
  - Usage: Star ratings, review scores, favorite indicators

## Implementation Flow

### Color Application Process:
1. **Start:** Theme Mode Selection
2. **Branch:** Route to appropriate theme color set
3. **Apply:** Component-specific color implementations
4. **Validate:** Universal colors maintain consistency
5. **Output:** Fully themed UI components

### Component Hierarchy:
```
Theme Selection
├── Navigation Bar Colors
├── Product Card Colors  
├── Form Input Colors
├── Button Colors
└── Universal Notification Colors (consistent across all)
```

---

## Technical Implementation Notes

### CSS Custom Properties Structure:
Each theme mode should define:
- Primary accent colors (theme-specific)
- Background colors (theme-specific)  
- Text colors (theme-specific)
- Surface colors (theme-specific)
- Border colors (theme-specific)
- Universal notification colors (constant)

### Color Inheritance:
- Universal colors never change between themes
- Component colors inherit from theme-specific variables
- Interactive states use appropriate contrast ratios
- Focus states maintain accessibility standards
